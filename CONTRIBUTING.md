# Contributing

Contributions are welcome. Please strictly adhere to the following guidelines.

## Issues

- **Bug Reports**: Clearly describe the issue, steps to reproduce, and expected behavior. Include log output if applicable.
- **Feature Requests**: Provide a concise rationale for the feature and its intended use case.

## Pull Requests

1. **Fork the repository** and create a feature branch.
2. **Code Style**: Maintain consistency with the existing codebase (standard JS/CSS patterns).
3. **Commit Messages**: Use clear, imperative subject lines (e.g., "Fix navigation bug", "Add tab support").
4. **Testing**: Ensure existing functionality remains unbroken.
5. **PR Description**: Reference relevant issues and summarize changes.
